jQuery Parallax Slider Plug-in Documentation

Requirements

Jquery 1.9 or newer

Optional jquery plugins:

  • jQuery easing plugin
  • jquery Color animation plugin

Intro

This plugin is designed to make it easy to create layered animations that also transition to new slides like a regular slideshow.

Parallax Scrolling is a cool visual effect that has been used for decades in games and media to give the illusion of 3d depth when only using flat 2d objects such as images & textures.

To create this effect, you need to animate each layer at a different rate of speed.

For example, if you had a landscape scene, things in the far off distance, such as a sky & mountain photo, would move more slowly then things closer to the camera, such plateau overlooking the valley, and trees in the foreground.

As a designer, you'll have to create these compelling images.

Search google for ideas of parallax scrolling animations when deciding how to design & build the various layers of your animation.

In addition to allow repeating textures to animate on multiple layers in the background of a slide, this plugin also allows smaller elements to be animated within the slide to do more then just parallax scrolling effects.

Hopefully, this plugin will help you achieve the desired effect for all your animations, but if it doesn't, let us know on github about your idea.

Getting Started With The Code

Assuming you have a properly structured HTML in your document, you can create a new instance of this parallax slider, with simple code like this in the <head> tag of your document:

<link rel="stylesheet" type="text/css" href="jquery.parallax-slider.css" />
<script src="example-files/jquery.1.9.1.min.js"></script>
<script src="jquery.parallax-slider.js"></script>
<script src="example-files/jquery.animate-colors.js"></script>
<script src="example-files/jquery.easing.1.3.js"></script> <script type="text/javascript">
$(document).ready(function(){
options={};
var p=$("#sliderContainer").parallaxSlider(options);
});
</script>

Global Options

The options supported by the parallaxSlider() function are fully documented at the top of the jquery.parallax-slider.js file.

Building Animations

In addition to configuring those global javascript options, it is also posible to configure individual slides, slide elements, and slide animated backgrounds using simple JSON (Javascript Object Notation) syntax to augment what is possible with regular CSS in order to support advanced animations without having to write complex javascript. The following describes the json structure allowed for each of the element types.

HTML Structure Documentation

Keep in mind that the nested structure of the HTML is an important part making sure your elements appear on the correct slide and are rigged correctly by this jquery plugin. The purpose of each element is described in the HTML comments below.

<!-- Outermost container must be given class name of zPSlideContainer -->
<div id="slideContainer" class="zPSlideContainer" style="width:100%; height:300px;position:relative; float:left; overflow:hidden;">

<!-- You must also have a container for the slide elements with the class name of zPSlideSlider -->
<div class="zPSlideSlider" style="float:left;width:100%;position:relative;">

<!-- Now you can create 1 or more slides with the class name zPSlide. Notice the data-config="" code below. This is how you change the animation options available for that specific slide. -->
<div class="zPSlide" data-config="{transition:'vertical',begin:{duration:5000}}" style="position:relative;width:100%; height:300px; left:0px; top:0px; background-color:#060;">

<!-- There are 3 kinds of special elements that can be inside a slide element. -->

<!-- The first is an animated background with the class name zPSlideAnimatedBackground. Notice the use of "data-background-image" attribute for specifying the image url instead of using background-image css. This is done so that the images can be "lazy loaded" the first time the slide is displayed. This means only your first slide will be loaded on first load, making for a fast loading experience. A few of these element moving at different durations will allow creating a parallax scrolling effect. You'll want to use repeating texture images if this is your goal. Note that it is also possible to animate a background horizontally and vertically at different speeds. More info is provided further down this documentation page. -->
<div class="zPSlideAnimatedBackground" data-config="{durationX:1000, animateX:136, animateY:136}" data-background-image="example-files/plasmad.png" style=" background-repeat:repeat; background-position:0px 0px; width:100%; height:300px;"></div>

<!-- The second element type is given a class name of zPSlideCenter. This element is a container for any additional elements you may want to animate that are not repeating texture backgrounds. It is required in order to ensure the relative positioning when resizing the browser. This allows for the slideshow to be stretched the full width of the browser, but no smaller then the width of this container element. -->
<div class="zPSlideCenter" style="width:600px; height:300px; margin:0 auto;position:relative;">

<!-- The third element type is given a class name of zPSlideElement. This is for smaller objects that you want to have animate in various ways with configurable delay, duration, easing and position. For example, you could a box that fades in while moving from top to bottom which doesn't start animating for 500 milliseconds through the use of CSS and the data-config json object. -->
<div class="zPSlideElement" style=" z-index:1001; left:300px; top:50px; width:100px; height:20px; background-color:#900;" data-config="{begin:{opacity:0, left:-100, top:0, duration:1000}, end:{opacity:0, left:0, top:0}}"></div>
<div class="zPSlideElement" style="z-index:1002; left:50px; top:200px; width:50px; height:100px; background-color:#090;" data-config="{begin:{opacity:0, left:-100, top:50, delay:500, duration:700}, end:{opacity:0, left:0, top:0}}"></div>
<div class="zPSlideElement" style="z-index:1003; left:150px; top:120px; width:70px; height:50px; background-color:#009;" data-config="{begin:{opacity:0, left:-100, top:200, delay:1000, duration:500}, end:{opacity:0, left:0, top:0}}"></div>

</div>

</div>

<!-- here you can have other slides with animated elements in them which don't appear until the slideshow is transitioned to the next slide. -->
<div class="zPSlide" data-config="{transition:'horizontal',begin:{duration:5000}}" style="position:relative;width:100%; height:300px; left:0px; top:0px; background-color:#006;">
<div class="zPSlideAnimatedBackground" data-config="{type:'transition',duration:400, animateX:'100%'}" data-background-image="example-files/plasma.png" style=" background-repeat:repeat; background-position:0px 0px; width:100%; margin-top:95px; height:121px;"></div>
<div class="zPSlideCenter" style="width:600px; height:300px; margin:0 auto;position:relative;">
<div class="zPSlideElement" style="z-index:1001; left:200px; top:200px; width:20px; height:50px; background-color:#900;" data-config="{begin:{opacity:0, left:-100, top:50, delay:1000, duration:1000}, end:{opacity:0, left:0, top:0}}"></div>
<div class="zPSlideElement" style="z-index:1002; left:0px; top:70px; width:350px; font-size:48px; line-height:48px; color:#FFF;" data-config="{begin:{opacity:0, left:-100, top:0, delay:500, duration:700}, end:{opacity:0, left:0, top:0}}">Any HTML can be animated</div>
<div class="zPSlideElement" style="z-index:1003; left:300px; top:130px; width:250px; height:50px; background-color:#009;" data-config="{begin:{opacity:0, left:-100, top:100, duration:500}, end:{opacity:0, left:0, top:0}}"></div>
</div>

</div>

</div>

<!-- Anywhere in your HTML document, you can also create buttons that can be registered for advancing the slideshow backwards and forwards. -->
<a href="#" id="sliderPrev" style="margin-top:-170px; margin-left:5%;">&lt;</a> <a href="#" id="sliderNext" style="margin-top:-170px; float:right;margin-right:5%;">&gt;</a>

<!-- You can also create a set of buttons which can be clicked to switch to a specific slide. Notice the "data-slide-index" attribute is provided, which identifies the slide that button will show. Note that we are not using the 0 index for this value, you must start with 1 for the first slide. -->
<div class="slideNavButtons">
<div style="width:330px;">
<a href="#" class="slideSetButton" data-slide-index="1">Slide1</a>
<a href="#" class="slideSetButton" data-slide-index="2">Slide2</a>
<a href="#" class="slideSetButton" data-slide-index="3">Slide3</a>
</div>
</div>

<script type="text/javascript">
$(document).ready(function(){
options={
setButtons:$(".slideSetButton")
};
var p=$("#sliderContainer").parallaxSlider(options);
});
</script>

JSON Object Documentation

Below we explain the custom JSON object structure supported on each class name. Most of the JSON object keys and values are optional. Adjust only the values you want to animate / change, and the global defaults will be used for the rest.

.zPSlideContainer

This element doesn't support any custom JSON data attributes.

.zPSlideSlider

This element doesn't support any custom JSON data attributes.

.zPSlide

This element supports an HTML Data attribute called "data-config", which can accept the following structure. Note all keys are optional including the data-config attribute itself. This works because parallaxSlider has default values for all options, which can be overrided with the parallaxSlider(options) function. Refer to the option documentation at the top of jquery.parallax-slider.js for more information.

{

transition:'horizontal', /* valid values are 'horizontal', 'vertical' or 'crossfade' */
begin:{

duration:5000, /* length of time the slide transition "in" animation. Measured in milliseconds. */
easing: 'easeInSine' /* easing used for the "in" animation. Note: you may need the jquery easing plug-in for most easing types. */

},
end:{

duration:5000, /* length of time the slide transition "out" animation. Measured in milliseconds. */
easing: 'easeOutSine' /* easing used for the "out" animation. Note: you may need the jquery easing plug-in for most easing types. */

}

}

Notice that there is no option for configuring the duration of the slide. The slide duration is automatically determined by waiting for all nested animations to complete their "in" animation AND duration timeouts. This makes saves you doing from having to do the math.

.zPSlideAnimatedBackground

This element supports an HTML Data attribute called "data-config", which can accept the following structure.

{

type:'transition', /* valid values are 'transition' or 'loop' | transition occurs once when the slide "in" animation begins. "loop" continues the animation infinitely until the slide "out" animation begins. */
duration:400, /* Set animation duration when using type:'transition'. Measured in milliseconds. */
durationX:1000, /* change the duration of the X background-position animation only. Measured in milliseconds. */
durationY:300, /* change the duration of the Y background-position animation only. Measured in milliseconds. */
animateX:136, /* change amount of movement for X background-position only. You can use pixel values (without px at the end) or a percentage. */
animateY:136, /* change amount of movement for Y background-position only. You can use pixel values (without px at the end) or a percentage. */

}

Note: You can control the original position of the background image before animation starts using CSS: background-position.

.zPSlideCenter

This element doesn't support any custom JSON data attributes.

.zPSlideElement

This element supports an HTML Data attribute called "data-config", which can accept the following structure.


{

begin:{

opacity:0, /* Set the opacity before the animation begins. At the end of the begin animation, the opacity will be the css "opacity" value on this element, which defaults to 1. */
left:-100, /* Set the "left" position relative to top left corner of zPSlideCenter before the animation begins. At the end of the begin animation, the "left" value is determined by the css "left" which must be set manually. */
top:0, /* Set the "top" position relative to top left corner of zPSlideCenter before the animation begins. At the end of the begin animation, the "top" value is determined by the css "top" which must be set manually. */
delay:500, /* Set the amount of milliseconds to wait before beginning the animation. */
duration:1000, /* The duration of the animation in milliseconds */
easing:'easeInOutBack' /* the easing function to use for this animation */

},
end:{

opacity:0, /* Set the opacity that will be used at the end of the animation. At the beginning, the opacity will be the css "opacity" value on this element, which defaults to 1. */
left:-100, /* Set the "left" position relative to top left corner of zPSlideCenter to animate to. At the end of the begin animation, the "left" value is determined by the css "left" which must be set manually. */
top:0, /* Set the "top" position relative to top left corner of zPSlideCenter to animate to. At the end of the begin animation, the "top" value is determined by the css "top" which must be set manually. */
delay:500, /* Set the amount of milliseconds to wait before beginning the animation. */
duration:1000, /* The duration of the animation in milliseconds */
easing:'easeInOutBack' /* the easing function to use for this animation */

}

}
